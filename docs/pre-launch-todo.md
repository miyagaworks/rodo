## 3. C 群: 法務・契約系（後日対応 / 並行進行可）

> **重要**: これらはコードに依存しないが、**SaaS として外部に提供する以上、本番運用開始前に揃えるべき**。
> 後日着手するが、shimoda 本番運用までに揃えることを推奨。

### 3.1 法務系（一般公開ページ）

| 項目 | 内容 | 想定対応 |
|---|---|---|
| **利用規約** | rodo.run のサービス利用規約。禁止事項・免責・準拠法・裁判管轄 | 法務テンプレートをベースに rodo 用にカスタマイズ |
| **プライバシーポリシー** | 個人情報の取得・利用目的・第三者提供・保管期間・問合せ先 | 顧客名・電話番号・住所・署名・GPS位置情報を扱うため必須 |
| **特商法表記** | 事業者名・住所・連絡先・販売価格・支払方法・返金条件 | SaaS 課金開始時に必須（無料β期間でも記載推奨） |
| **Cookie ポリシー** | NextAuth セッション Cookie・解析 Cookie の説明 | プライバシーポリシーに含めても可 |

### 3.2 shimoda との契約・覚書

| 項目 | 内容 | 想定対応 |
|---|---|---|
| **利用契約** | サービス利用範囲、料金（プロト期間中は無料？）、契約期間、解約条件 | 契約書ドラフト作成 |
| **責任分界点** | 障害時の責任範囲、補償上限、SLA未達時の扱い | 契約書に明記 |
| **データ所有権** | shimoda 業務データの所有者は shimoda、rodo 側は預かるだけ | 契約書に明記 |
| **解約時のデータ返還** | 解約時のデータエクスポート方法、保管期間、削除証明 | 手順書とセット |
| **個人情報の取扱合意** | shimoda の顧客情報を rodo 側で扱うことの合意（個人情報保護法 27条委託先監督） | 委託契約書 or 利用契約に組込 |
| **広告塔協力** | shimoda の社名・ロゴ・事例を 2社目営業に使う合意 | ヒアリング E-3 で確認済み、覚書化 |

### 3.3 運用合意

| 項目 | 内容 | 想定対応 |
|---|---|---|
| **uptime SLA の合意** | 月次稼働率の目標値（99% / 99.5% / 99.9%）、計画メンテ枠 | shimoda と口頭合意 + 議事録 |
| **メンテナンス時間枠** | 月次・四半期メンテの曜日・時間帯（例: 第3日曜深夜） | shimoda 業務に支障の少ない枠 |
| **障害連絡先（双方向）** | 宮川 → shimoda、shimoda → 宮川 の連絡経路と一次対応者 | P2-06 と連動 |
| **データバックアップ頻度の合意** | 日次・週次・保管期間 30日 等 | P2-05 と連動 |

### 3.3A 障害時の業務継続合意（最重要、`rollback-criteria.md` 7B 節参照）

| ID | 合意項目 | 必要時期 |
|---|---|---|

| AG-02 | 障害発生時、宮川からの「紙運用復帰」連絡があれば即時切替に応じる | 切替日 1 週間前まで |

| AG-07 | 本番切替日から最初の 1 週間は、宮川が朝夕にステータス確認連絡を入れる | 切替日 1 週間前まで |
