## 3. このセッションで判明した重要な仕様（次回への注意点）

### 3.1 「出発 ODO の前回値表示」は placeholder のみ

**動作**: 新規出動時、`/dispatch/new` で出発 ODO 欄に「000000km」が薄いグレーで表示されることがある。

**仕様確定（バグではない）**:
- `lastReturnOdo` は **placeholder（薄いグレー表示）** として渡されている
- `value` への自動入力は **実装されていない**
- 表示色: 薄いグレー `#9CA3AF`（未入力ガイド）/ 濃い青 `#1C2948`（入力済）

**「000000km」になる条件**:
1. 隊員に **車両（vehicleId）が割り当てられていない** → API すら呼ばれず `lastReturnOdo=null` → placeholder=0
2. 車両はあるが、その車両でまだ完了案件が無い → API が `lastReturnOdo: null` → placeholder=0
3. API エラー → 同上

**実機検証時の前提**:
- B-18 以降の検証では、隊員に **車両を割り当ててから** 始めること
- 隊員設定（`/settings`）で `User.vehicleId` を確認・設定
- 車両未割当のまま検証して「000000」が出ても **それは仕様** であってバグではない

**関連ファイル**（参考）:
- `app/api/dispatches/last-return-odo/route.ts:1-50`
- `components/dispatch/DispatchClient.tsx:307-326`（fetch）, `:411-412`（placeholder 算出）, `:1024-1030`（OdoDialInput 呼び出し）
- `components/common/OdoDialInput.tsx:686-690`（value=null時の表示）, `:772`（文字色切替）
