## 1. このドキュメントの目的

shimoda.rodo.run の本番運用中に発生する障害（システムダウン・データ異常・ユーザー報告）への一次対応フローを定義する。

「真夜中に障害が起きたとき、誰が誰に何分以内に何を伝えるか」が明確でないと、初動が遅れて shimoda 業務に長時間影響する。

---

## 2. 障害の重大度分類

| 重大度 | 定義 | 例 | 対応開始目標 |
|---|---|---|---|
| **致命（P0）** | shimoda 業務が完全に停止 | 全ユーザーログイン不可 / 出動データ書き込み不可 / データ漏洩疑い | 検知から **15 分以内** |
| **重大（P1）** | 主要機能の一部が停止 | 写真アップロード不可 / PDF 生成失敗 / 一部ユーザーのみログイン不可 | 検知から **1 時間以内** |
| **中（P2）** | 機能は動くが不便 | 表示の遅延 / 一部画面でエラー表示 / 軽微な計算ずれ | 検知から **当日中** |
| **軽（P3）** | UI 表記揺れ・誤字 | ボタン位置のずれ / 軽微な誤字 | 次回リリース |

## 4. 障害検知から対応完了までのフロー

### 4.1 検知

検知経路は以下の 3 つ:

1. **自動検知**（Sentry / Vercel Analytics アラート → Slack/メール通知）
2. **shimoda 側からの通報**（電話・LINE）
3. **宮川による定期確認**（朝夕の状態チェック）

### 4.2 一次対応（検知から 15 分以内）

| # | 担当 | アクション |
|---|---|---|
| 1 | 宮川（受信者） | 障害内容を「重大度」で分類（P0〜P3） |
| 2 | 宮川 | P0 / P1 の場合、即座に shimoda 側責任者へ電話連絡 |
| 3 | 宮川 | Sentry / Vercel ログを確認、影響範囲を特定 |
| 4 | 宮川 | 暫定回避策の有無確認（例: 別ブラウザで試す、再ログイン等） |
| 5 | 宮川 | shimoda 側に「対応中」「予想復旧時間」「業務継続可否」を伝達 |

### 4.3 復旧対応

| 状況 | 対応 |
|---|---|
| 直近デプロイ起因と判明 | Vercel Instant Rollback（Level 1）→ 1〜3 分で復旧 |
| インフラ障害（Vercel / Neon） | プロバイダー側ステータス確認、復旧待ち、shimoda へ進捗連絡 |
| コード起因で 30 分以上かかる | ロールバック判断（`docs/rollback-criteria.md` に従う） |
| データ起因（DB 直接修正必要） | バックアップから復旧 or 手動修正、必ず作業ログを残す |

### 4.4 復旧確認

- 障害再現条件で動作確認
- shimoda 側に「復旧完了」を電話連絡
- shimoda 側で実機確認、業務再開承認を取得

### 4.5 事後対応

| 期限 | アクション | 担当 |
|---|---|---|
| 当日中 | 障害発生時刻・原因仮説・影響範囲・復旧時刻の一次記録 | 宮川 |
| 24 時間以内 | ポストモーテム作成（`docs/postmortems/YYYY-MM-DD-<件名>.md`） | 宮川 |
| 48 時間以内 | shimoda へ正式報告書送付（再発防止策含む） | 宮川 |
| 1 週間以内 | 再発防止策の実装またはチケット化 | 宮川 + 実装CC |

## 5. ポストモーテムテンプレート

```markdown
# ポストモーテム: <障害件名>

## 概要
- 発生日時: YYYY-MM-DD HH:MM 〜 HH:MM
- 重大度: P0 / P1 / P2 / P3
- 影響範囲: <ユーザー数 / 機能>
- 検知経路: 自動検知 / ユーザー通報 / 宮川確認

## タイムライン
- HH:MM 検知
- HH:MM 一次対応開始
- HH:MM 原因特定
- HH:MM 復旧対応開始
- HH:MM 復旧確認
- HH:MM shimoda へ完了連絡

## 直接原因
<技術的な原因>

## 根本原因（5 Whys）
1. なぜ A が起きたか？
2. なぜ B が起きたか？
...

## 業務影響
<shimoda 側の業務にどう影響したか、定量的に>

## 対応内容
<実施したアクション>

## 良かった点
<迅速に動けた点・想定通りに機能した予防策>

## 改善点
<次回より早く対応するための改善>

## 再発防止策
- [ ] アクション1（担当・期限）
- [ ] アクション2（担当・期限）
```
